# README – AWS Week 1 Terminology Quick Reference & MCQ Quiz

## Overview

This project contains a visual poster and an interactive MCQ quiz for **AWS Week 1** revision.

It is designed to help learners quickly review important AWS and DevOps terms from Week 1 and then test their understanding using a browser-based quiz.

## How to Use the Quiz

Open the quiz file in any browser:

```text
aws-week-1-mcq-quiz-new-colors.html
```

Then:

1. Read each question carefully.
2. Select one answer.
3. Complete all 25 questions.
4. Click **Submit Quiz**.
5. Review your score and explanations.
6. Click **Reattempt & Shuffle** to practice again with shuffled questions and answers.

## Week 1 Topics Covered

| Topic | What You Learn |
|---|---|
| AWS Account Security | Root user, MFA, billing alerts, budget monitoring |
| IAM Basics | Users, groups, roles, and principals |
| IAM Policies | AWS managed policies, customer managed policies, JSON policy structure |
| S3 | Buckets, objects, object keys, bucket ARN, object ARN |
| EC2 | Virtual servers and read-only access |
| Billing | Billing dashboard and billing read-only permissions |
| STS | Temporary credentials and role assumption |
| Switch Role | IAM user assuming an IAM role |
| GitHub OIDC | Secure GitHub Actions access to AWS without long-lived keys |
| GitHub Actions | Workflow, job, step, runner, and OIDC permissions |
| Least Privilege | Give only required permissions |

## Important Terms

| Term | Meaning |
|---|---|
| IAM | Identity and Access Management |
| MFA | Multi-Factor Authentication |

## Best Practice Reminder

```text
Security first.
Use MFA.
Avoid root user for daily work.
Use IAM groups and roles.
Use least privilege.
Use temporary credentials when possible.
Avoid long-lived AWS access keys.
Clean up unused AWS resources after labs.
```
