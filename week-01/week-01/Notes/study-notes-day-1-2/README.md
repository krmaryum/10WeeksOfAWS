# AWS Day 1 & Day 2 — README

## Overview

This study package covers the first two days of AWS learning.

- **Day 1:** AWS Cloud Foundations
- **Day 2:** IAM Users, Groups, Roles & Policies

---

## Day 1 — AWS Cloud Foundations

Day 1 explains the basic foundation of AWS Cloud.

### Topics Covered

| Topic | Summary |
|---|---|
| AWS Learning Path | Start with cloud basics, security, core services, and hands-on practice |
| Cloud Computing | On-demand IT resources over the internet |
| Core AWS Services | Focus first on IAM, EC2, S3, VPC, RDS, CloudWatch, and Lambda |
| Cloud Service Models | IaaS, PaaS, and SaaS |
| Why AWS | Scalability, reliability, flexibility, security, and cost-effectiveness |
| AWS Global Infrastructure | Regions, Availability Zones, Edge Locations, Local Zones, and Wavelength Zones |
| Shared Responsibility Model | AWS secures the cloud; customer secures resources in the cloud |
| AWS Account Setup | Create and secure an AWS account |
| Billing Basics | Use budgets, alerts, and delete unused resources |
| Root User vs IAM User | Root is for account-level tasks; IAM user is for daily work |
| MFA | Adds extra security to AWS login |
| Well-Architected Pillars | Operational Excellence, Security, Reliability, Performance Efficiency, Cost Optimization, and Sustainability |

---

## Day 2 — IAM Users, Groups, Roles & Policies

Day 2 focuses on AWS IAM and security basics.

### Topics Covered

| Topic | Summary |
|---|---|
| IAM | Identity and Access Management controls AWS access |
| Authentication | Who are you? |
| Authorization | What are you allowed to do? |
| IAM User | Identity for a person or workload |
| IAM Group | Collection of users with shared permissions |
| IAM Role | Temporary permissions for trusted entities like EC2 or Lambda |
| IAM Policy | JSON document that allows or denies AWS actions |
| Managed Policy | Reusable policy managed by AWS or customer |
| Inline Policy | Policy attached directly to one identity |
| JSON Policy Structure | Version, Statement, Effect, Action, Resource, and Condition |
| Least Privilege | Give only required permissions |
| Permission Boundary | Sets maximum permissions |
| IAM Best Practices | Use MFA, roles, groups, least privilege, and regular permission review |
